# Custom Language Virtual Machine

This is a small virtual machine and interpreter project guided by *Crafting Interpreters*.  
The goal of the project is to understand how interpreters work and what makes a programming language executable.

## Purpose

I started this project to explore the connection between high-level programming languages and low-level program behavior. Instead of only using programming languages as tools, I want to understand how a language is read, represented, and executed by an interpreter.

## Topics I Want to Learn

- Lexical analysis and parsing
- Syntax design
- Abstract syntax trees
- Bytecode and instruction execution
- Stack-based virtual machines
- Runtime behavior
- Memory management
- Efficient program design
- Error handling in interpreters

## Current Status

Currently in early development.

Planned first milestones:

- [ ] Define a minimal language syntax
- [ ] Build a scanner/tokenizer
- [ ] Implement a parser
- [ ] Represent expressions/statements internally
- [ ] Execute simple expressions
- [ ] Add variables and control flow
- [ ] Explore bytecode execution and VM design
